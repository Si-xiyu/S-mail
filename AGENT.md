# SmartMail AI 协作说明

本文件适用于 Claude Code、Codex 以及其他 AI 编码助手。成员使用 AI 工具前，请先让工具阅读本文件、`项目架构与技术选型.md` 以及 `doc/todo/2026年实训要求.pdf`。

## 项目定位

SmartMail 是一个 Web 版智能邮件系统实训项目。

- 基础能力：注册登录、发送邮件、接收/拉取邮件、收件箱、已发送、附件、已读未读、删除、搜索、同步等。
- 智能能力：以可插拔方式接入 AI，例如邮件摘要、回复草稿、优先级预测、标签建议、垃圾邮件识别。
- 首版不优先实现真实 SMTP/POP3，采用 HTTP REST 自定义上层邮件协议。
- 假设所有用户都在同一个服务器下，不考虑跨邮件服务器投递。

## 技术路线

- 前端：Vue 3 + Vite + TypeScript + Element Plus + Pinia + Axios。
- 后端：Spring Boot 3 + Java 17 + MyBatis-Plus + MySQL 8 + JWT。
- AI 服务：Python FastAPI，可选启动，不应影响基础邮件功能。
- 存储：邮件正文入库，附件存磁盘，数据库只保存附件元信息。

## 仓库结构

```text
frontend/     Vue 前端项目
backend/      Spring Boot 主业务后端
agent/        Python AI Agent 服务，可选
docs/         接口、设计、测试、报告等项目文档
doc/          原始需求资料和已有 prompt 文档
storage/      本地附件存储目录
```

后端按业务模块组织：

```text
auth/         登录、注册、JWT
user/         用户资料和邮箱地址
mail/         邮件主数据、发送、详情
mailbox/      收件箱、已发送、草稿、垃圾箱、已读未读
attachment/   附件上传、下载、元信息
label/        标签
search/       搜索和筛选
ai/           AI 门面和结果管理
internal/     仅供 Agent 调用的内部接口
common/       通用响应、异常、安全、工具类
```

## 开发原则

1. 先完成基础邮件闭环，再做 AI 加分项。
2. 后端基础功能不能依赖 AI 服务；Agent 关闭时系统仍应可用。
3. 邮件主数据和用户邮箱视图要分离：`mail_message` 保存邮件内容，`mailbox_item` 保存用户视角下的文件夹、已读、星标、删除状态。
4. 附件不要直接存数据库，保存到磁盘目录，数据库保存路径、文件名、MIME、大小等元信息。
5. 前端只调用 Spring Boot API，不直接调用 Agent。
6. Agent 不直接访问数据库，通过后端 `internal` 接口读取邮件和保存 AI 结果。

## AI 工具使用约定

- 修改代码前先阅读相关模块，不要凭空创建与现有结构冲突的目录或命名。
- 不要删除、移动或重写他人正在修改的文件，除非任务明确要求。
- 不要把前端代码放进后端目录，也不要把后端业务逻辑写到前端。
- 新增接口时同步更新 `docs/api/` 中的接口说明。
- 修改数据库结构时同步更新 `docs/design/` 中的数据表设计。
- 遇到不确定的技术选择，优先遵循 `项目架构与技术选型.md`。
- 不要提交密钥、Token、数据库密码或大模型 API Key。

## 文件生成约定

- 不要在项目根目录随意生成临时文件、报告、草稿、日志或一次性脚本。
- 新增文件必须放入与用途匹配的既有目录：
  - 前端代码：`frontend/`
  - 后端代码：`backend/`
  - Agent 服务：`agent/`
  - 正式项目文档：`docs/`
  - 原始资料、计划书、课程资料：`doc/`
  - 本地运行数据：`storage/`
- 临时验证文件优先放入 `.tmp/`，并且不要提交。
- 如确实需要新增根目录文件，必须先说明用途，并确认没有更合适的既有目录。
- 不要创建 `test.md`、`note.md`、`result.txt`、`summary.md`、`temp.*` 等无明确归属的根目录文件。

## 推荐开发顺序

1. 初始化后端 Spring Boot 工程、数据库连接和统一响应。
2. 完成用户注册、登录、JWT 鉴权。
3. 完成邮件发送、收件箱、已发送、邮件详情。
4. 完成附件上传下载。
5. 完成已读未读、删除、搜索、标签。
6. 初始化前端 Vue 工程并接入上述接口。
7. 最后接入 Agent，实现摘要、回复草稿、优先级和标签建议。
