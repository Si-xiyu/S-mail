# SmartMail

SmartMail 是一个 Web 版智能邮件系统实训项目，目标是在同一服务器用户之间实现基础邮件收发能力，并以可插拔方式接入 AI 智能功能。

本项目采用前后端分离架构，首版不优先实现真实 SMTP/POP3 协议，而是基于 HTTP REST 自定义上层邮件协议，先完成稳定的邮件业务闭环，再逐步加入智能能力。

## 项目目标

- 邮箱注册、登录和用户身份校验。
- 发送邮件、接收邮件、拉取邮件。
- 收件箱、已发送、草稿箱、垃圾箱等基础邮箱视图。
- 附件上传、下载和独立存储。
- 已读/未读、删除、搜索、标签等邮件管理能力。
- 邮件正文存数据库，附件存磁盘，数据库保存附件元信息。
- 可插拔 AI 功能，例如邮件摘要、回复草稿、优先级预测、标签建议、垃圾邮件识别。

## 技术路线

| 模块 | 技术 |
| --- | --- |
| 前端 | Vue 3、Vite、TypeScript、Element Plus、Pinia、Axios |
| 后端 | Spring Boot 3、Java 17、MyBatis-Plus、MySQL 8、JWT |
| AI 服务 | Python、FastAPI、Pydantic、LLM SDK |
| 存储 | MySQL、本地磁盘附件存储 |
| 通信 | HTTP REST API |

```text

mailbox/        收件箱、已发送、草稿、垃圾箱、已读未读

label/          标签

internal/       仅供 Agent 调用的内部接口
common/         通用响应、异常处理、安全配置、工具类
```

## 推荐开发顺序

1. 初始化 Spring Boot 后端工程，完成数据库连接、统一响应和异常处理。
2. 完成注册、登录、JWT 鉴权和当前用户接口。
3. 设计并实现邮件核心表：用户、邮件主表、收件人表、邮箱条目表、附件表。
4. 完成发送邮件、收件箱、已发送、邮件详情。
5. 完成附件上传下载、已读未读、删除、搜索、标签。
6. 初始化 Vue 前端工程，接入登录、邮箱列表、邮件详情、写信等页面。
7. 接入 Python Agent，实现邮件摘要、回复草稿、优先级和标签建议。
8. 补充测试文档、演示数据和答辩材料。

## 协作约定

- 修改代码前先阅读 `CLAUDE.md` 和 `项目架构与技术选型.md`。
- 新增或修改接口时，同步更新 `docs/api/`。
- 修改数据库结构时，同步更新 `docs/design/`。
- 不提交密钥、Token、数据库密码或大模型 API Key。
- AI 服务是增强能力，不能成为基础邮件系统的强依赖。
- 前端只调用 Spring Boot API，不直接访问数据库或 Agent。
- Agent 通过后端 `internal` 接口读取邮件和保存 AI 结果。
