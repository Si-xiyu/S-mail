› 我们一共有六个人，应该怎么分工？


• 建议按 2 前端 + 2 后端 + 1 AI/算法 + 1 组长/集成测试 分工。这样最符合你们这个项目的结构，也方便并行开发。

  6 人分工
  ├─ 组长 / 架构 / 集成
  ├─ 前端 1：基础页面与主布局
  ├─ 前端 2：邮件操作与 AI 交互
  ├─ 后端 1：用户认证与通用基础
  ├─ 后端 2：邮件核心业务
  └─ AI / 测试 / 文档

  组长 / 架构 / 集成

  负责整体节奏和最终集成。

  主要任务：

  - 维护需求拆解和开发计划。
  - 确定接口规范、数据库设计、模块边界。
  - 协调前后端接口变更。
  - 负责 Git 分支、合并、冲突处理。
  - 整理最终答辩 PPT 和演示流程。
  - 把控 docs/ 文档完整性。

  适合技术和沟通能力都比较强的人。

  前端 1：基础页面与主布局

  负责前端基础框架和邮箱主界面。

  主要任务：

  - 初始化 Vue 3 + Vite + TypeScript 项目。
  - 配置 Vue Router、Pinia、Axios。
  - 登录页、注册页。
  - 主布局：左侧菜单、顶部搜索、用户信息区。
  - 收件箱、已发送、垃圾箱等基础页面框架。
  - 和后端 1 对接登录注册接口。

  对应目录：

  frontend/src/router/
  frontend/src/stores/
  frontend/src/layouts/
  frontend/src/views/auth/
  frontend/src/views/mail/

  前端 2：邮件操作与 AI 交互

  负责具体邮件功能页面。

  主要任务：

  - 邮件列表组件。
  - 邮件详情组件。
  - 写邮件弹窗或写邮件页面。
  - 附件上传、附件下载展示。
  - 已读未读、删除、搜索、标签交互。
  - AI 摘要、回复草稿、优先级提示面板。

  对应目录：

  frontend/src/components/
  frontend/src/api/
  frontend/src/views/mail/

  后端 1：用户认证与通用基础

  负责后端基础设施。

  主要任务：

  - 初始化 Spring Boot 项目。
  - 配置 MySQL、MyBatis-Plus。
  - 统一响应格式。
  - 全局异常处理。
  - 用户注册、登录。
  - 密码加密。
  - JWT 鉴权。
  - 当前用户信息接口。
  - 基础权限拦截。

  对应模块：

  backend/.../auth/
  backend/.../user/
  backend/.../common/

  后端 2：邮件核心业务

  负责邮件系统主功能。

  主要任务：

  - 邮件发送接口。
  - 邮件主表、收件人表、邮箱条目表。
  - 收件箱、已发送列表。
  - 邮件详情。
  - 已读未读。
  - 删除到垃圾箱。
  - 搜索和筛选。
  - 附件上传下载。
  - 标签功能。

  对应模块：

  backend/.../mail/
  backend/.../mailbox/
  backend/.../attachment/
  backend/.../label/
  backend/.../search/

  AI / 测试 / 文档

  负责智能功能和测试材料。

  主要任务：

  - 初始化 agent/ FastAPI 服务。
  - 实现邮件摘要接口。
  - 实现回复草稿接口。
  - 实现优先级和标签建议。
  - 写 AI 插件设计文档。
  - 编写测试计划、测试用例、测试报告。
  - 准备演示数据。
  - 如果 AI 来不及，先做规则版垃圾邮件识别或规则优先级判断。

  对应目录：

  agent/
  docs/design/ai-design.md
  docs/test/
  docs/report/

  推荐开发顺序：

  第一阶段：
  后端 1 搭认证，后端 2 设计邮件表，前端 1 搭页面框架，组长定接口。

  第二阶段：
  打通登录注册、发邮件、收件箱、已发送、邮件详情。

  第三阶段：
  加入附件、删除、已读未读、搜索、标签。

  第四阶段：
  接入 AI 摘要、回复草稿、优先级建议，补测试和答辩材料。

  最关键的是：后端 1 和后端 2 先一起把数据库表和接口文档定下来，前端再按接口开发。否则前端容易先写死假数据，后面联调成本会变高。
